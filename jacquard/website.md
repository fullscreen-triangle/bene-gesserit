Here is the checklist for the website

1. Motive : I wanted this framework to have some web presence for two reasons : to broadcast the framework to collaborators and for inviting funding

2. Structure : It should be a dark themed website by default, and it should be professional and scientific in nature 

3. Content :  It should contain a page for each paper, whereby you extract the important details in the paper for website visitors.. these should not be short bullet points, but well designed scientific papers... that is, do not just link the pdfs, extract the content and generate charts using d3.js from the results Each paper page should contain around 75% of the content in the paper
